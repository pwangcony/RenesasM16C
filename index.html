M16C F/W Guide

시스템 하드웨어 제약사항 요약

본 문서는 고객사 요청에 따라 작성된 커스텀 시스템 가이드입니다. 해당 시스템은 MCU의 내부 리소스를 배제하고 오직 외부 인터페이스를 통한 메모리 구성을 채택하고 있습니다. 펌웨어 업데이트 설계 시 이 제약사항을 최우선으로 고려해야 합니다.

💻

Microcontroller

M16CM30302SPFP

  • 주의: Internal RAM 사용 안 함
  • 주의: Internal Flash 사용 안 함
  • Processor Mode: Microprocessor Mode
🧮

External RAM

R1LP0108ESN

  • Type: 1M SRAM
  • Organization: 128K word x 8-bit
  • 역할: 데이터 저장 및 F/W 업데이트 코드 실행 공간
💾

External Flash

MX29F800CBT

  • Type: 8M-BIT [1M x 8 / 512K x 16]
  • Feature: Boot Sector Flash
  • 역할: 메인 애플리케이션 저장소

💡 핵심 해결 과제

내부 메모리가 없으므로, 외부 플래시(MX29F800CBT)를 지우고 쓰는 동안에는 플래시에서 코드를 읽어 실행할 수 없습니다. 따라서 플래시 제어 루틴은 반드시 부팅 시 외부 RAM(R1LP0108ESN)으로 복사된 후 RAM에서 실행되어야 합니다.

펌웨어 업데이트 절차 (RAM Base)

ROM(Flash)에 쓰기/지우기 작업을 수행하는 동안에는 해당 ROM 공간에서 명령어를 Fetch할 수 없습니다. 따라서 업데이트 루틴은 반드시 외부 RAM으로 복사되어 실행되어야 합니다.

업데이트 시작 시, 플래시 메모리(MX29F800CBT)를 제어하는 함수들(Erase, Program, Check Status)을 External RAM(R1LP0108ESN) 영역으로 복사해야 합니다.

// Pseudo-code example void copy_flash_routines_to_ram(void) { uint8_t *src = (uint8_t *)&flash_erase_sector; // Address in Flash uint8_t *dst = (uint8_t *)RAM_ROUTINE_START; // Address in Ext RAM uint16_t size = &flash_routines_end - &flash_erase_sector; for(int i=0; i<size; i++) { dst[i] = src[i]; } }

플래시 작업 중 인터럽트가 발생하여 Flash 영역의 Vector 테이블을 참조하게 되면 시스템이 정지(Crash)됩니다. 반드시 인터럽트를 비활성화(asm(" fcli");) 하거나, 인터럽트 벡터 테이블 자체를 RAM으로 재배치해야 합니다. 그 후 RAM에 복사된 업데이트 메인 함수로 프로그램 카운터(PC)를 이동시킵니다.

MX29F800CBT 데이터시트에 명시된 Command Sequence에 따라 Data Bus를 통해 명령어를 전송합니다.

  • Sector Erase: 555H(AAH) -> 2AAH(55H) -> 555H(80H) -> 555H(AAH) -> 2AAH(55H) -> Sector Address(30H)
  • Byte/Word Program: 555H(AAH) -> 2AAH(55H) -> 555H(A0H) -> Address(Data)

주의: 외부 버스 타이밍(Wait state 설정)이 플래시 칩의 요구사양을 충족하는지 버스 컨트롤러 레지스터를 재확인하십시오.

Data Polling(DQ7) 또는 Toggle Bit(DQ6)를 모니터링하여 Erase/Program 작업의 완료 여부를 확인합니다. 모든 작업이 완료되고 데이터 무결성(Checksum/CRC) 확인이 끝나면, Watchdog Timer를 발생시키거나 Reset 핀을 제어하여 시스템을 하드웨어 리셋합니다. 이후 새 펌웨어로 부팅됩니다.

플래시 메모리 동작 성능 요약

선정된 외부 Flash Memory (MX29F800CBT)의 스펙 기반 예상 작업 소요 시간 분포입니다. F/W 업데이트 시 전체 소요 시간 산정 및 Watchdog Timer 설정에 참고하십시오.

작업별 예상 시간 (Typical)

  • Sector Erase (64KB)

    약 0.7초 / 섹터당

  • Chip Erase (Total)

    약 9초 전체

  • Word Program (16-bit)

    약 11 µs / 단어당

분석: 전체 펌웨어 이미지 크기가 512KB라고 가정할 때, Erase 작업에 대부분의 시간이 소요되며(약 5.6초), 프로그래밍(쓰기)에는 상대적으로 적은 시간(약 0.3초)이 소요됩니다. 사용자가 대기 시간을 지루해하지 않도록 진행률(Progress) 표시에 이 가중치를 반영하는 것이 좋습니다.
